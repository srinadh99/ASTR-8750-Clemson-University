Section 01

Why Prediction Intervals?

A regression model outputs ŷ = f(x), a point estimate. But how far off might it be? In medicine, finance, or safety-critical systems, knowing the range of plausible outcomes is as important as the estimate itself.

Classical approaches (t-intervals, bootstrap, Bayesian credible intervals) all require distributional assumptions or are only asymptotically valid. Conformal prediction provides exact finite-sample coverage with no such requirements.

💡
The Promise

For any pre-chosen α ∈ (0,1) and any base model, conformal prediction constructs an interval C(x) such that P(Y ∈ C(X)) ≥ 1 − α — exactly, not approximately.

Section 02

Split Conformal Algorithm for Regression

Split data into 3 parts

Training set (fit model), calibration set (compute quantile), test set (evaluate). Typical split: 60% / 20% / 20%.

Train any regression model f̂ on training data

Linear regression, gradient boosting, neural network — conformal prediction is model-agnostic.

Compute nonconformity scores on calibration set

For each calibration example i: sᵢ = |yᵢ − f̂(xᵢ)|. These measure how wrong the model is.

Find quantile threshold q̂

q̂ = Quantile({s₁…sₙ}, level = ⌈(n+1)(1−α)⌉/n). This is the (1−α)-th order statistic, adjusted for the new test point.

Form prediction interval at test time

C(x*) = [f̂(x*) − q̂, f̂(x*) + q̂]. The true y* lies in this interval with probability ≥ 1−α.

C(x*) = [ f̂(x*) − q̂ , f̂(x*) + q̂ ] where q̂ = Quantile_{ ⌈(n+1)(1-α)⌉/n }( {|yᵢ - f̂(xᵢ)|}ᵢ₌₁ⁿ )
Section 03

Basic CP: Constant-Width Intervals

scikit-learnnumpymatplotlib
basic_conformal_regression.py
import numpy as np
import matplotlib.pyplot as plt
from sklearn.ensemble import GradientBoostingRegressor
from sklearn.model_selection import train_test_split

# ── Generate heteroskedastic data: y = sin(x) + x·ε ─────────────────
np.random.seed(0)
n = 1000
X = np.random.uniform(0, 10, n).reshape(-1,1)
y = np.sin(X.flatten()) + 0.1 * X.flatten() * np.random.randn(n)

# ── 3-way split ─────────────────────────────────────────────────────
X_tr, X_tmp, y_tr, y_tmp = train_test_split(X, y, test_size=0.4, random_state=0)
X_cal, X_te, y_cal, y_te = train_test_split(X_tmp, y_tmp, test_size=0.5, random_state=0)

# ── Train model ─────────────────────────────────────────────────────
model = GradientBoostingRegressor(n_estimators=200, max_depth=3)
model.fit(X_tr, y_tr)

# ── Calibration scores: absolute residuals ───────────────────────────
scores = np.abs(y_cal - model.predict(X_cal))

def get_qhat(scores, alpha):
    n = len(scores)
    level = np.ceil((n + 1) * (1 - alpha)) / n
    return np.quantile(scores, min(level, 1.0), interpolation='higher')

# ── Build interval at α = 0.10 (90% coverage) ───────────────────────
alpha  = 0.10
q_hat = get_qhat(scores, alpha)
y_hat = model.predict(X_te)
lower, upper = y_hat - q_hat, y_hat + q_hat

coverage = ((y_te >= lower) & (y_te <= upper)).mean()
avg_width = (upper - lower).mean()
print(f"q̂ = {q_hat:.4f}")
print(f"Coverage = {coverage:.3f}  (target ≥ {1-alpha:.2f})")
print(f"Avg width = {avg_width:.4f}")
Output
q̂ = 0.4832
Coverage = 0.914 (target ≥ 0.90)
Avg width = 0.9664
Figure 1 — Conformal Prediction Intervals on Heteroskedastic Data
Band width is constant despite increasing noise in data (x > 5)
⚠️
Problem with constant-width intervals

Notice the data spreads more for large x (heteroskedastic). Basic CP gives the same-width interval everywhere — too wide on the left (wasteful), too narrow on the right (risky in practice). Section 4 fixes this.

Section 04

Adaptive CP: Normalised Residuals

Replace the score |y−ŷ| with the normalised residual |y−ŷ| / σ̂(x), where σ̂(x) is a locally-estimated noise level. Now the interval width adapts to each input.

adaptive_conformal_regression.py
from sklearn.ensemble import GradientBoostingRegressor

# ── Train a variance (spread) estimator on squared residuals ─────────
resid_sq = (y_tr - model.predict(X_tr)) ** 2
var_est = GradientBoostingRegressor(n_estimators=100)
var_est.fit(X_tr, resid_sq)

def sigma(X):
    return np.sqrt(np.maximum(var_est.predict(X), 1e-6))

# ── Normalised calibration scores ─────────────────────────────────
scores_norm = np.abs(y_cal - model.predict(X_cal)) / sigma(X_cal)
q_norm = get_qhat(scores_norm, alpha)

# ── Adaptive test intervals ────────────────────────────────────────
y_hat  = model.predict(X_te)
sig_te = sigma(X_te)
lo_ad, hi_ad = y_hat - q_norm * sig_te, y_hat + q_norm * sig_te

cov_ad = ((y_te >= lo_ad) & (y_te <= hi_ad)).mean()
w_basic   = (2 * q_hat)
w_adaptive = (hi_ad - lo_ad).mean()

print(f"Adaptive coverage : {cov_ad:.3f}")
print(f"Basic width (const): {w_basic:.4f}")
print(f"Adaptive avg width : {w_adaptive:.4f}")
Output
Adaptive coverage : 0.906
Basic width (const): 0.9664
Adaptive avg width : 0.8831
Figure 2 — Adaptive vs. Basic Prediction Intervals
Adaptive band widens as noise increases with x

Section 06

The Coverage Guarantee — Formal Proof

Let (X₁,Y₁),…,(Xₙ,Yₙ),(X_{n+1},Y_{n+1}) be exchangeable. Define scores sᵢ = s(Xᵢ,Yᵢ). Let s_{n+1} = s(X_{n+1},Y_{n+1}) be the hypothetical test score.

P( Y_{n+1} ∈ C_α(X_{n+1}) ) = P( s_{n+1} ≤ q̂ ) ≥ 1 − α

Proof sketch: By exchangeability, the rank of s_{n+1} among {s₁,…,sₙ,s_{n+1}} is uniform on {1,…,n+1}. Define q̂ as the ⌈(n+1)(1−α)⌉-th smallest score. Then:

P( s_{n+1} ≤ q̂ ) = ⌈(n+1)(1−α)⌉ / (n+1) ≥ 1 − α □

The upper bound follows because ⌈x⌉ ≤ x + 1, giving P(s_{n+1} ≤ q̂) ≤ 1 − α + 1/(n+1).

📐
Key assumptions

Only exchangeability is needed — not IID, not normality, not any parametric family. In practice, exchangeability is implied by IID sampling but is strictly weaker.

Section 07

Cross-Conformal Prediction

Split conformal "wastes" calibration data. Cross-conformal uses K-fold CV to reuse every point for both training and calibration.

cross_conformal.py
from sklearn.model_selection import KFold

kf = KFold(n_splits=5, shuffle=True, random_state=42)
all_scores = np.zeros(len(X_tr))  # store OOF residuals

for fold_idx, (tr_idx, val_idx) in enumerate(kf.split(X_tr)):
    m = GradientBoostingRegressor(n_estimators=200)
    m.fit(X_tr[tr_idx], y_tr[tr_idx])
    all_scores[val_idx] = np.abs(y_tr[val_idx] - m.predict(X_tr[val_idx]))

# Use all OOF residuals as one big calibration pool
q_cv = get_qhat(all_scores, alpha=0.10)

# Fit final model on all training data
final_model = GradientBoostingRegressor(n_estimators=200)
final_model.fit(X_tr, y_tr)

y_cv = final_model.predict(X_te)
cov_cv = ((y_te >= y_cv - q_cv) & (y_te <= y_cv + q_cv)).mean()
print(f"Cross-conformal coverage: {cov_cv:.3f}")
print(f"Cross-conformal width   : {2*q_cv:.4f}")
Output
Cross-conformal coverage: 0.902
Cross-conformal width : 0.9418
⚠️
Note on cross-conformal

Cross-conformal is not guaranteed to have exact coverage (only asymptotically). For finite samples, split conformal is theoretically preferred. Cross-conformal is useful when n is small and data efficiency matters.

Section 08

Pitfalls and Best Practices

⚠️
Pitfall 1: Distribution shift at test time

If test X has different distribution than calibration X, the coverage guarantee breaks. Use weighted conformal or monitor your data drift.

⚠️
Pitfall 2: Leaking test data into calibration

If any test point was used (even indirectly) in fitting the nonconformity score model, coverage guarantees are invalidated. Strict data splits are mandatory.

✅
Tip 1: Use MAPIE for production

pip install mapie — implements split-conformal, CV+, and CQR out of the box with sklearn-compatible API.

✅
Tip 2: Prefer CQR for heteroskedastic data

Conformalized Quantile Regression (CQR) by Romano et al. (2019) uses a quantile regressor as the base model and gives the best adaptive intervals.
